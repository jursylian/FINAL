# План фаз и чек‑листы

Навигация: [README.md](README.md) → [03-backend-architecture.md](03-backend-architecture.md)

Фазы описаны строго в заданном порядке. После backend‑auth сразу идет frontend‑auth.

## Фаза 0. Установка библиотек и настройка проекта
Подзадачи:
- Инициализация backend/frontend
- Установка зависимостей
- Базовая структура каталогов
- Подключение MongoDB
- Настройка `.env`

Чек‑лист готовности:
- [ ] Backend запускается
- [ ] Frontend запускается
- [ ] MongoDB подключен
- [ ] `.env` заполнен корректно

## Фаза 1. Модели данных (минимум для auth)
Подзадачи:
- Модель `User` с индексами
- Валидация полей
- Подготовка к хэшированию пароля

Чек‑лист готовности:
- [ ] Есть модель `User`
- [ ] Уникальные индексы `email` и `username`
- [ ] Пароль не возвращается в ответах
- [ ] Email валидируется, пароль имеет минимальную длину

## Фаза 2. Backend Auth
Подзадачи:
- `POST /auth/register`
- `POST /auth/login`
- JWT‑подпись
- `auth` middleware
- `GET /auth/me`

Чек‑лист готовности:
- [ ] Регистрация создает пользователя
- [ ] Логин возвращает JWT
- [ ] `GET /auth/me` защищен и возвращает профиль без пароля

## Фаза 3. Frontend Auth
Подзадачи:
- Страницы регистрации/логина
- Получение и хранение токена
- `Authorization: Bearer ...` для запросов
- Protected routes
- Обработка ошибок

Чек‑лист готовности:
- [ ] Можно зарегистрироваться и войти
- [ ] Токен сохраняется и используется
- [ ] Защищенные маршруты недоступны без токена
- [ ] Ошибки 400/401/409 показываются пользователю

## Фаза 4. Остальные модели данных
Подзадачи:
- `Post`, `Like`, `Comment`, `Follow`, `Notification`
- Опционально `Message`
- Индексы и связи

Чек‑лист готовности:
- [ ] Все модели добавлены
- [ ] Уникальные индексы для лайков и подписок
- [ ] Связи и ссылки описаны

## Фаза 5. Профиль
Подзадачи:
- `GET /users/:id`
- `PATCH /users/:id`
- Загрузка аватара
- UI профиля

Чек‑лист готовности:
- [ ] Профиль возвращается без пароля
- [ ] Профиль редактируется только владельцем
- [ ] Аватар загружается через FormData

## Фаза 6. Посты
Подзадачи:
- CRUD постов
- Лента
- Посты пользователя
- Загрузка изображений
- UI ленты/создания/детали

Чек‑лист готовности:
- [ ] Пост создается с изображением
- [ ] Лента отображается
- [ ] Детальная страница поста работает

## Фаза 7. Лайки и комментарии
Подзадачи:
- Toggle like
- Добавление и список комментариев
- UI лайков и комментариев

Чек‑лист готовности:
- [ ] Лайк не дублируется
- [ ] Комментарий добавляется
- [ ] Список комментариев отображается

## Фаза 8. Поиск и Explore
Подзадачи:
- Поиск пользователей
- Explore (случайные посты)
- UI поиска и explore

Чек‑лист готовности:
- [ ] Поиск возвращает корректные результаты
- [ ] Explore возвращает случайные посты

## Фаза 9. Подписки
Подзадачи:
- Follow/unfollow
- Followers/following
- UI подписок

Чек‑лист готовности:
- [ ] Подписка не дублируется
- [ ] Списки followers/following корректны

## Фаза 10. Уведомления
Подзадачи:
- Список уведомлений
- Отметка прочитанными
- UI уведомлений

Чек‑лист готовности:
- [ ] Уведомления создаются на like/comment/follow
- [ ] Уведомления можно пометить прочитанными

## Фаза 11. Чат Socket.io (опционально)
Подзадачи:
- Подключение Socket.io
- События и комнаты
- Хранение сообщений
- UI чата

Чек‑лист готовности:
- [ ] Сообщения доставляются в реальном времени
- [ ] История сообщений хранится
